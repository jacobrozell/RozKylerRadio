GitHub limits (plain English; see URLs for exact wording)

REPOSITORY / GIT (all plans)
  https://docs.github.com/en/repositories/creating-and-managing-repositories/repository-limits
  - Single file in normal Git: enforced max 100 MB (use Git LFS above that).
  - Push size: max 2 GB per push.
  - GitHub recommends keeping repos small (on-disk under ~1 GB ideal, under ~5 GB strongly recommended).

LARGE FILES (plain Git vs LFS)
  https://docs.github.com/en/repositories/working-with-files/managing-large-files/about-large-files-on-github
  - Over 50 MB: warning.
  - Over 100 MB: push blocked unless using Git LFS.

GIT LFS (if you use it for WAVs etc.)
  https://docs.github.com/en/repositories/working-with-files/managing-large-files/about-git-large-file-storage
  https://docs.github.com/en/billing/managing-billing-for-your-products/managing-billing-for-git-large-file-storage/about-billing-for-git-large-file-storage
  - GitHub Free: 2 GB max per LFS file; includes quota (see billing doc for current included storage/bandwidth).
  - Docs also note Git LFS is not wired into GitHub Pages the way small static files are; for Actions-based Pages deploy, checkout with LFS and materialize files before upload so the published site contains real media, not pointer text.

GITHUB PAGES (published site)
  https://docs.github.com/en/pages/getting-started-with-github-pages/github-pages-limits
  - Published site size: max 1 GB.
  - Soft bandwidth: 100 GB/month.
  - Deploy timeout: 10 minutes.

THIS POOL ON DISK (radio/Renders)
